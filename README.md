## 브랜치
- GitHub 브랜치 전략 사용
    - 브랜치 이름은 기능을 상세히 나타내도록 작성
    - merge지 전에는 PR을 통해 어떤 기능을 구현했는지 상세하게 작성
    - Git Push 는 수시로 진행하기 
    - main 브랜치는 언제든지 배포 가능한 상태로 유지
- 브랜치 종류
    - main
    - develop
        - 상세 기능 구분하여 구현
        - loginPage, recordPage..
